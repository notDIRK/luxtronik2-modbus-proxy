# Quick Task 260409-m53: Add Parameter Backup Button and Backup Sensor Summary

**One-liner:** Backup Parameters button reads all 1,126 Luxtronik parameters via luxtronik.__new__ pattern and saves timestamped JSON to /config/luxtronik2_backups/, with a TIMESTAMP sensor showing the last backup time.

## What Was Built

**LuxtronikBackupButton** (`button.py`):
- `EntityCategory.CONFIG` — appears under device Configuration section in HA
- `async_press()` acquires `coordinator._lock` (ARCH-03) then dispatches `_sync_backup()` to executor (ARCH-02)
- `_sync_backup()` uses `luxtronik.Luxtronik.__new__` pattern (ARCH-01) for a fresh read of all parameters
- JSON format: `{timestamp, host, parameters: {index: {name, type, raw_value, display_value}}}`
- Saves to `{ha_config}/luxtronik2_backups/YYYY-MM-DD_HH-MM-SS.json`
- Creates `persistent_notification` with count, filename, timestamp after success
- Stores metadata in `hass.data[DOMAIN][f"{entry_id}_last_backup"]` for sensor

**LuxtronikLastBackupSensor** (`sensor.py`):
- `EntityCategory.DIAGNOSTIC`, `SensorDeviceClass.TIMESTAMP`
- Extends `SensorEntity` directly (not `CoordinatorEntity`) — reads `hass.data`, not poll data
- `native_value` parses stored ISO timestamp string to `datetime.datetime`
- `extra_state_attributes` exposes `{"filename": ...}`
- `async_added_to_hass()` subscribes to `{DOMAIN}_backup_complete` dispatcher signal via `async_on_remove`
- Updates immediately when backup completes without waiting for next coordinator poll

**Platform registration:**
- `"button"` added to `PLATFORMS` in `__init__.py`
- `BACKUP_DIR = "luxtronik2_backups"` added to `const.py`

**Translations** (strings.json, en.json, de.json):
- `entity.button.backup_parameters.name`: "Backup Parameters" / "Parameter-Sicherung"
- `entity.sensor.last_backup.name`: "Last Backup" / "Letzte Sicherung"
