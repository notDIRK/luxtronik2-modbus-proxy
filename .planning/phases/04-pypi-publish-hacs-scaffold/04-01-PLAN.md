  <action>
Create `.github/workflows/publish.yml` with the following exact content (per D-02):

```yaml
name: Publish to PyPI

on:
  push:
    tags:
      - "v*"

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - name: Install build frontend
        run: pip install build
      - name: Build distributions
        run: python -m build
      - uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/

  publish:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: pypi
      url: https://pypi.org/p/luxtronik2-modbus-proxy
    permissions:
      id-token: write
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist/
      - uses: pypa/gh-action-pypi-publish@release/v1
```

Key points enforced:
- Triggers ONLY on `v*` tags (not on push to main) — avoids republish anti-pattern
- `id-token: write` is set at job level, not workflow level — minimal privilege scope (per Research anti-patterns)
- Uses `pypa/gh-action-pypi-publish@release/v1` (NOT `@master` which is sunset)
- Two-job pattern: build artifacts are uploaded then downloaded — clean separation
- Environment name `pypi` must match the pending publisher configuration on pypi.org

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions -> PyPI | OIDC token exchange crosses trust boundary; compromised workflow could publish malicious packages |
| PyPI package -> consumers | Published package is consumed by pip install; supply chain trust |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | publish.yml OIDC | mitigate | Use `environment: pypi` which requires GitHub environment protection rules; `id-token: write` only on publish job, not workflow-level |
| T-04-02 | Tampering | Build artifacts between jobs | mitigate | Use `actions/upload-artifact@v4` and `actions/download-artifact@v4` (GitHub-hosted, integrity-checked); two-job pattern isolates build from publish |
| T-04-03 | Elevation of Privilege | publish.yml triggers | mitigate | Trigger ONLY on `v*` tags (not on push to main); prevents accidental or unauthorized publishes from branch pushes |
| T-04-04 | Information Disclosure | pyproject.toml URLs | accept | URLs point to public GitHub repo and GitHub Pages docs — all public information by design |
</threat_model>
