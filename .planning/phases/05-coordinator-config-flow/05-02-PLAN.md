      Note: BenPru domain is `"luxtronik"` (not `"luxtronik2_modbus_proxy"`)

      try:
          await self.hass.async_add_executor_job(self._test_connection, host, DEFAULT_PORT)

          return self.async_create_entry(
              title=host,
              data={CONF_HOST: host, "port": DEFAULT_PORT, "poll_interval": DEFAULT_POLL_INTERVAL},
          )

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> config flow | User-supplied host IP string from HA UI form |
| config flow -> Luxtronik controller | TCP connection attempt to user-supplied IP on port 8889 |
| __init__.py -> hass.data | Coordinator stored in shared HA data namespace |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Spoofing | config_flow.async_step_user | accept | Config flow runs under HA's authenticated session; only authenticated HA admins can add integrations. No additional auth needed. |
| T-05-05 | Tampering | config_flow._test_connection | mitigate | Host input stripped of whitespace via `host = user_input[CONF_HOST].strip()`. Voluptuous schema enforces str type. Host is used only as TCP connection target, not in shell commands or URLs. |
| T-05-06 | Information Disclosure | strings.json | accept | Error messages reveal only that port 8889 is used (already documented publicly). No credentials or internal paths exposed. |
| T-05-07 | Denial of Service | config_flow._test_connection | mitigate | Connection test runs in executor thread (not event loop). luxtronik library has built-in socket timeout. Slow/unresponsive host blocks only the executor thread, not the HA event loop. |
</threat_model>
