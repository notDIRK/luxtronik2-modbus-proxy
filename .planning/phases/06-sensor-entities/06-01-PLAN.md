<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| coordinator.data -> sensor entity | Raw integer values from Luxtronik controller cross into HA entity display |
| luxtronik library from_heatpump() | Untrusted raw ints from controller processed by library conversion functions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | coordinator.data raw values | accept | Read-only sensor entities; no user input reaches coordinator.data. Values originate from local network controller behind LAN. Malicious values would display wrong temps but cannot execute code. |
| T-06-02 | Information Disclosure | sensor entity values | accept | Heat pump temperatures and status are non-sensitive operational data. No PII or credentials involved. Entity values visible only to authenticated HA users. |
| T-06-03 | Denial of Service | 1,377 entity registration at startup | accept | One-time registration cost; disabled entities have zero per-poll overhead. HA handles large entity counts routinely (e.g., Z-Wave networks). |
| T-06-04 | Spoofing | from_heatpump() return values | accept | Library function is deterministic and stateless; no external input beyond the raw int already validated by coordinator as int type. |
</threat_model>
