Create `.github/workflows/docs.yml` (per D-09) with the official Material for MkDocs deployment pattern:

```yaml
name: docs
on:
  push:
    branches:
      - main
permissions:
  contents: write
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Configure Git Credentials
        run: |
          git config user.name github-actions[bot]
          git config user.email 41898282+github-actions[bot]@users.noreply.github.com
      - uses: actions/setup-python@v5
        with:
          python-version: 3.x
      - run: echo "cache_id=$(date --utc '+%V')" >> $GITHUB_ENV
      - uses: actions/cache@v4
        with:
          key: mkdocs-material-${{ env.cache_id }}
          path: ~/.cache
          restore-keys: |
            mkdocs-material-
      - run: pip install mkdocs-material==9.7.6 mkdocs-static-i18n==1.3.1
      - run: mkdocs gh-deploy --force
```

Run `mkdir -p .github/workflows` before writing.

Key details:
- Pin action versions: `actions/checkout@v4`, `actions/setup-python@v5`, `actions/cache@v4`
- Pin MkDocs library versions in pip install: `mkdocs-material==9.7.6 mkdocs-static-i18n==1.3.1`
- `permissions: contents: write` is required for gh-pages push
- Weekly cache key via `%V` ISO week number
- Trigger only on push to `main` branch
