## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config.yaml -> RegisterMap | User-provided parameter names from config file are resolved against the database |
| CLI input -> list-params | Search term from CLI argument filters output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | config.yaml parameter names | mitigate | `resolve_parameter_names` validates every name against NAME_TO_INDEX; unknown names cause fail-fast ValueError at startup before any network activity |
| T-02-05 | Denial of Service | Extremely long parameter list in config.yaml | accept | Each extra parameter adds one dict entry (~100 bytes); even all 1,126 params < 200 KB. No amplification risk. |
| T-02-06 | Information Disclosure | list-params CLI output | accept | Outputs parameter names and types from the open-source luxtronik library. No private data. No IP addresses or credentials in output. |
</threat_model>
