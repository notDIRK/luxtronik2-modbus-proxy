       ```python
       # Update input registers for visibilities (read-only, mapped at offset 1000).
       for address in self._register_map.all_visibility_addresses():
           entry = self._register_map.get_visibility_entry(address)
           if entry is None:
               continue
           # Wire address = lux_index + 1000, so lux_index = wire_address - 1000
           lux_index = address - 1000
           visi = lux.visibilities.visibilities.get(lux_index)
           if visi is None:
               continue
           # Visibilities are Unknown type; value is 0 or 1 integer
           raw_value = visi.to_heatpump(visi.value) if visi.value is not None else 0
           cache.update_input_values(address, [int(raw_value) if raw_value is not None else 0])
       ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | Expanded register space (1355 input, 5001 holding) | accept | All registers contain heat pump sensor data (temperatures, modes); no PII or secrets. Mitigation at network level (local network only, per CLAUDE.md). |
| T-02-02 | Tampering | New visibility registers | mitigate | All 355 visibility entries are read-only (writable=False in RegisterEntry). Input registers use FC4 (read-only function code). No write path exists for visibility data. |
| T-02-03 | Denial of Service | Large datablock allocation (5001 holding + 1355 input) | accept | Total memory ~15 KB. Trivial for any deployment target. No amplification vector. |
</threat_model>
