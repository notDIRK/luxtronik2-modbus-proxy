<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Modbus client -> ProxyHoldingDataBlock | Untrusted write values from any LAN Modbus client |
| Proxy -> Luxtronik controller | Writes forwarded to physical hardware |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ProxyHoldingDataBlock.async_setValues | mitigate | Validate every write value against register_map.validate_write_value before accepting; return ExcCodes.ILLEGAL_VALUE for out-of-range values |
| T-02-02 | Tampering | ProxyHoldingDataBlock | mitigate | enable_writes=False by default; all writes rejected unless explicitly enabled in config |
| T-02-03 | Denial of Service | Luxtronik controller NAND flash | mitigate | Write rate limiting enforced in Plan 03's polling engine (1 write per 60s per register) |
| T-02-04 | Denial of Service | Luxtronik TCP port 8889 | mitigate | Connect-per-call pattern with immediate disconnect; never hold socket open between cycles (PROTO-05) |
| T-02-05 | Spoofing | write_queue | accept | Queue is in-process memory only; no external access; LAN attacker would need Modbus client access which is accepted for local network proxy |
</threat_model>
