<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Modbus client -> server -> datablock | Write requests from untrusted LAN clients reach the write queue |
| Write queue -> Luxtronik controller | Validated writes forwarded to physical hardware |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | polling_engine._drain_and_write | mitigate | Per-register write rate limiting: max 1 write per write_rate_limit (default 60s) per register address; excess writes logged as WARNING and dropped |
| T-03-02 | Denial of Service | polling_engine.run_forever | mitigate | try/except around _poll_cycle prevents crash on transient upstream errors; cache marked stale on failure |
| T-03-03 | Tampering | write queue | mitigate | Writes already validated by ProxyHoldingDataBlock.async_setValues (Plan 02); polling engine trusts queue contents |
| T-03-04 | Denial of Service | Modbus server port | accept | Standard Modbus TCP port 502; LAN-only deployment; no authentication (Modbus protocol has none) |
| T-03-05 | Information Disclosure | structlog output | mitigate | DEBUG level shows raw values; INFO level shows event types and register addresses but not raw values; production default is INFO |
</threat_model>
